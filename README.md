# アプリ名
タイトル: cospul ~あらゆる衣装の情報交換サイト~


# 開発環境
 - 言語: Ruby,JavaScript
 - フレームワーク Ruby on Rails, jQuery
 - DB Mysql
 - サーバー AWS
# 内容
ダンスインストラクターの仕事で発表会の生徒用の衣装探しに苦労したので、<br>
衣装の情報等がすぐにわかる情報サイトを作りました。<br><br>
ユーザー登録なしであらゆる衣装のサンプルをタグ検索等で見る事が出来、<br>
ユーザー登録すると投稿はもちろん、<br>
楽天市場商品検索apiを使い、商品サイトへのリンクを貼る事が出来、<br>
いいね機能を使いお気に入りをまとめて見ることも出来ます。<br><br>
主にjQueryを使った簡単なゲームもいくつか実装しています。

# 何ができるのか？
cospulは、以下のことができます。
- コーディネートを写真や詳細、複数のタグ付けなどをして投稿/編集/削除することができます。

- コーディネートに紐づいた、１つ１つの服の商品詳細を投稿でき、

- アクセサリーなど詳細カラムにないものも追加して投稿することができます

- コーディネートのアイテム登録画面やトップページ右上のプラスボタンを押した後に出てくるカートボタンを押すと、<br>
  楽天市場商品検索apiを使った商品検索を非同期通信で行うことができ、<br>
  アイテム詳細画面にURLを貼ると商品詳細画面ではボタンが表示され、商品のページに遷移することができます。 

- 商品についているタグをクリックすると同じタグのついた記事を一覧で見ることができます。

- トップページではタグのついた記事の検索をすることができ、インクリメンタルサーチでタグの候補を出すことができます。 

- ログインするといいねすることができ、いいねした記事はサイドバーより一覧で見ることができます。

- 右上のプラスボタンを押すとテトリスや簡単のゲームを複数遊ぶことができます。

# 工夫した点、苦労した点

1. 作ったことのないタグやいいねなどのSNSによくある機能への挑戦。<br>
TECK::EXPERTカリキュラム内でコメント機能などの実装は行なったことがありましたが、<br>
いいね機能、タグ機能等は作ったことがないので挑戦しました。<br>
実際に作ってみると多対多のレコードの関係やいいねをしたのかしてないのかの判定など思った以上に仕組みが複雑で実装に苦労しましたが、<br>
タグの検索やインクリメンタルサーチ、いいねしたものの一覧など実装すればするほど奥深く勉強になりました<br>

2. 楽天商品検索apiの導入
  　初めて知らないapiを導入するにあたり、<br>
  apiの知識も少なくrails +楽天商品検索apiの記事も思ったような記事が見つからず苦労しました。<br>
  理解や知識が増えてくるにつれて、apiキーやパラーメターの使い方など少しづつ理解することが出来、<br>
  最終的には非同期で商品情報を表示することにも成功しました。


3. １つのformで２つのモデルを扱う機能
初めて１つのformで２つのモデルを扱う機能を作ったので実装に苦労しました。<br>
始めはパラメーターの付け方や１つ１つのコードの意味がわからず理解に苦しみましたが、<br>
この経験を生かしTECK::EXPERTでの最終課題では再現度の高い登録方法を作ることができ、<br>
その後さらに個人アプリで写真を複数登録するとボタンのサイズが変わる機能や、<br>
ネストするレコードを削除する機能もつけることができるようになりました。


4. jQueryの導入
jQueryのライブラリーや機能を積極的に導入したのですが、<br>
製作当初はライブラリーの導入の仕方がいまいち理解できず苦労しました。<br>
少しづつjQueryの理解が出来るようになり、<br>
製作途中からはライブラリーを使っていた物をより自分好みにするために<br>
新しくコードを打ちながら製作し直したものもありました。
